## 1. JWT认证机制

### 1.1 JWT工具类 (JwtUtil.java)

JWT（JSON Web Token）是一种无状态的认证方式，适合前后端分离架构。

```java
@Component
public class JwtUtil {

    // 密钥字符串（至少32位）
    private final String SECRET_STRING = "homestay_backend_secret_key_fixed_12345678";
    private final Key key = Keys.hmacShaKeyFor(SECRET_STRING.getBytes());
    
    // Token有效期：24小时
    private final long expiration = 1000 * 60 * 60 * 24;

    /**
     * 生成JWT Token
     * @param userId 用户ID
     * @param username 用户名
     * @param role 角色（USER/HOST/ADMIN）
     * @return JWT Token字符串
     */
    public String generateToken(Long userId, String username, String role) {
        return Jwts.builder()
                .setSubject(username)              // 主题：用户名
                .claim("userId", userId)           // 自定义声明：用户ID
                .claim("role", role)               // 自定义声明：角色
                .setIssuedAt(new Date())           // 签发时间
                .setExpiration(new Date(System.currentTimeMillis() + expiration)) // 过期时间
                .signWith(key)                     // 签名密钥
                .compact();
    }

    /**
     * 从Token中提取用户ID
     */
    public Long getUserIdFromToken(String token) {
        Claims claims = Jwts.parserBuilder()
                .setSigningKey(key)
                .build()
                .parseClaimsJws(token)
                .getBody();

        // 类型转换处理（Integer → Long）
        Object userId = claims.get("userId");
        if (userId instanceof Integer) {
            return ((Integer) userId).longValue();
        }
        return (Long) userId;
    }

    /**
     * 从Token中提取用户名
     */
    public String getUsernameFromToken(String token) {
        return Jwts.parserBuilder()
                .setSigningKey(key)
                .build()
                .parseClaimsJws(token)
                .getBody()
                .getSubject();
    }
}
```

**关键点说明**：
1. **密钥管理**：使用固定字符串生成密钥，确保服务重启后Token仍然有效
2. **Token结构**：包含用户名（subject）、用户ID、角色三个核心信息
3. **有效期**：24小时，过期后需要重新登录
4. **类型转换**：处理Integer和Long的兼容性问题

### 1.2 JWT过滤器 (JwtFilter.java)

JWT过滤器拦截所有HTTP请求，验证Token并设置认证信息。

```java
public class JwtFilter extends OncePerRequestFilter {

    private final JwtUtil jwtUtil;
    private final UserRepository userRepository;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain)
            throws ServletException, IOException {

        // 1. 跳过静态资源和公开接口
        if (shouldNotFilter(request)) {
            filterChain.doFilter(request, response);
            return;
        }

        // 2. 获取Authorization Header
        String header = request.getHeader("Authorization");

        // 3. 验证Token格式：Bearer {token}
        if (header != null && header.startsWith("Bearer ")) {
            try {
                // 提取Token（去掉"Bearer "前缀）
                String token = header.substring(7);

                // 4. 解析Token
                Claims claims = Jwts.parserBuilder()
                        .setSigningKey(jwtUtil.getKey())
                        .build()
                        .parseClaimsJws(token)
                        .getBody();

                String username = claims.getSubject();
                String role = claims.get("role", String.class);

                if (username != null && role != null) {
                    // 5. 检查用户状态
                    User user = userRepository.findByUsername(username);
                    if (user == null) {
                        SecurityContextHolder.clearContext();
                        filterChain.doFilter(request, response);
                        return;
                    }
                    
                    // 检查账号是否被禁用
                    if ("DISABLED".equals(user.getStatus())) {
                        SecurityContextHolder.clearContext();
                        response.setStatus(HttpServletResponse.SC_FORBIDDEN);
                        response.setContentType("application/json;charset=UTF-8");
                        response.getWriter().write("{\"msg\":\"账号已被禁用，请联系管理员\"}");
                        return;
                    }
                    
                    // 6. 格式化角色（添加ROLE_前缀）
                    // Spring Security的hasRole()方法需要ROLE_前缀
                    String formattedRole = role.toUpperCase().startsWith("ROLE_") ?
                            role.toUpperCase() : "ROLE_" + role.toUpperCase();

                    // 7. 创建权限列表
                    List<SimpleGrantedAuthority> authorities =
                            Collections.singletonList(new SimpleGrantedAuthority(formattedRole));

                    // 8. 构建认证对象
                    UsernamePasswordAuthenticationToken authentication =
                            new UsernamePasswordAuthenticationToken(username, null, authorities);

                    // 9. 设置到SecurityContext（关键步骤）
                    SecurityContextHolder.getContext().setAuthentication(authentication);
                }
            } catch (Exception e) {
                // Token异常（过期、签名错误等）
                System.out.println("JWT认证失败: " + e.getMessage());
                SecurityContextHolder.clearContext();
            }
        }

        // 10. 继续执行过滤器链
        filterChain.doFilter(request, response);
    }
    
    /**
     * 判断是否跳过JWT验证
     */
    @Override
    protected boolean shouldNotFilter(HttpServletRequest request) {
        String path = request.getRequestURI();
        
        // 跳过静态资源（图片、CSS、JS等）
        if (path.matches(".+\\.(jpg|jpeg|png|gif|svg|ico|css|js|woff|woff2|ttf|eot)$")) {
            return true;
        }
        
        // 跳过上传目录
        if (path.startsWith("/uploads/") || path.startsWith("/house/")) {
            return true;
        }
        
        return false;
    }
}
```

**认证流程图**：
```
客户端请求
    ↓
提取Authorization Header
    ↓
验证Token格式（Bearer {token}）
    ↓
解析Token（提取用户名、角色）
    ↓
检查用户状态（是否存在、是否禁用）
    ↓
格式化角色（添加ROLE_前缀）
    ↓
创建认证对象
    ↓
设置到SecurityContext
    ↓
继续执行后续过滤器
```

```java
    result.put("role", user.getRole());

}
```

## 2. 订单日期冲突检测

### 2.1 问题描述

同一房源不能在同一时间段被多次预订。需要检测新订单的日期范围是否与已有订单冲突。

### 2.2 冲突检测算法

**日期冲突的判断条件**：
```
两个订单冲突 ⟺ 
  (新订单入住日期 < 已有订单退房日期) AND 
  (新订单退房日期 > 已有订单入住日期)
```

**图示**：
```
已有订单：  |-------|
新订单1：        |-------|  ✗ 冲突
新订单2：|---|                ✓ 不冲突
新订单3：              |---|  ✓ 不冲突
```

### 2.3 SQL实现 (OrderMapper.java)

```java
@Mapper
public interface OrderMapper extends BaseMapper<Order> {

    /**
     * 查询指定日期范围内的冲突订单数量
     * 
     * 逻辑说明：
     * 1. 只检查未取消的订单（status != 'CANCELLED'）
     * 2. 使用NOT排除不冲突的情况：
     *    - check_out_date <= checkIn：已有订单在新订单之前结束
     *    - check_in_date >= checkOut：已有订单在新订单之后开始
     * 3. 如果count > 0，说明存在冲突
     */
    @Select("SELECT COUNT(*) FROM `order` " +
            "WHERE house_id = #{houseId} " +
            "AND status != 'CANCELLED' " +
            "AND NOT (check_out_date <= #{checkIn} OR check_in_date >= #{checkOut})")
    int countConflictingOrders(@Param("houseId") Long houseId,
                                @Param("checkIn") LocalDate checkIn,
                                @Param("checkOut") LocalDate checkOut);
}
```

### 2.4 业务层实现 (OrderService.java)

```java
@Service
public class OrderService {

    private final OrderMapper orderMapper;

    /**
     * 检查房源在指定日期是否可用
     * @return true-可用，false-不可用
     */
    public boolean isHouseAvailable(Long houseId, LocalDate checkIn, LocalDate checkOut) {
        int count = orderMapper.countConflictingOrders(houseId, checkIn, checkOut);
        return count == 0;  // count为0表示无冲突
    }

    /**
     * 创建订单
     */
    public Order createOrder(Order order) {
        // 1. 生成订单号：HS + 时间戳 + 随机数
        if (order.getOrderNo() == null || order.getOrderNo().isEmpty()) {
            order.setOrderNo("HS" + System.currentTimeMillis() + (int)(Math.random() * 1000));
        }
        
        // 2. 设置创建时间
        if (order.getCreatedTime() == null) {
            order.setCreatedTime(LocalDateTime.now());
        }
        
        // 3. 插入数据库
        orderMapper.insert(order);
        return order;
    }
}
```

## 3. 智能推荐算法

### 3.1 推荐系统架构

本系统采用**混合推荐策略**，结合多种推荐算法：

```
推荐系统架构
├── 协同过滤推荐（Collaborative Filtering）
│   ├── 基于用户的协同过滤（User-Based CF）
│   └── 基于物品的协同过滤（Item-Based CF）
├── 内容推荐（Content-Based）
│   ├── 基于城市偏好
│   ├── 基于价格偏好
│   └── 基于房源特征
└── 热门推荐（Popularity-Based）
    ├── 评分排序
    └── 订单量排序
```

### 3.2 推荐服务接口 (RecommendationController.java)

```java
@RestController
@RequestMapping("/api/recommend")
public class RecommendationController {

    @Autowired
    private RecommendationService recommendationService;

    /**
     * 为当前用户推荐房源
     * @param limit 推荐数量
     * @param method 推荐方法：hybrid(混合), cf(协同过滤), content(内容推荐)
     */
    @GetMapping("/for-me")
    public ResponseEntity<?> recommendForMe(
            @RequestParam(defaultValue = "10") Integer limit,
            @RequestParam(defaultValue = "hybrid") String method,
            Principal principal) {
        
        // 1. 未登录用户返回热门房源
        if (principal == null) {
            List<RecommendationDTO> popular = recommendationService.getPopularHouses(limit);
            return ResponseEntity.ok(popular);
        }

        // 2. 获取当前用户
        String username = principal.getName();
        User user = userRepository.findByUsername(username);
        
        // 3. 获取个性化推荐
        List<RecommendationDTO> recommendations = recommendationService.recommendForUser(
                user.getId(), limit, method);
        
        // 4. 如果推荐结果不足，补充热门房源
        if (recommendations == null || recommendations.isEmpty()) {
            recommendations = recommendationService.getPopularHouses(limit);
        } else if (recommendations.size() < limit) {
            // 补充热门房源（去重）
            List<RecommendationDTO> popular = recommendationService.getPopularHouses(limit * 2);
            Set<Long> existingIds = recommendations.stream()
                    .map(RecommendationDTO::getHouseId)
                    .collect(Collectors.toSet());
            
            for (RecommendationDTO p : popular) {
                if (!existingIds.contains(p.getHouseId())) {
                    recommendations.add(p);
                    if (recommendations.size() >= limit) break;
                }
            }
        }
        
        return ResponseEntity.ok(recommendations);
    }

    /**
     * 推荐相似房源
     */

    public ResponseEntity<?> recommendSimilar(
```

### 3.3 协同过滤算法详解

#### 3.3.1 算法原理

**协同过滤（Collaborative Filtering）** 是推荐系统中最常用的算法之一，基本思想是：
- **用户相似性**：喜欢相似物品的用户是相似的
- **物品相似性**：被相似用户喜欢的物品是相似的

**基于用户的协同过滤（User-Based CF）**：
```
1. 找到与目标用户相似的其他用户
2. 推荐这些相似用户喜欢但目标用户未浏览的房源
```

**基于物品的协同过滤（Item-Based CF）**：
```
1. 找到与用户已浏览房源相似的其他房源
2. 推荐这些相似房源
```

#### 3.3.2 相似度计算

**余弦相似度（Cosine Similarity）**：
```
similarity(A, B) = cos(θ) = (A · B) / (||A|| × ||B||)

其中：
- A · B：向量点积
- ||A||：向量A的模长
- ||B||：向量B的模长
```

**示例**：
```
用户A浏览记录：[房源1, 房源2, 房源3]
用户B浏览记录：[房源2, 房源3, 房源4]

向量表示：
用户A：[1, 1, 1, 0]  (房源1,2,3,4的浏览情况)
用户B：[0, 1, 1, 1]

相似度计算：
A · B = 0×1 + 1×1 + 1×1 + 0×1 = 2
||A|| = √(1² + 1² + 1² + 0²) = √3
||B|| = √(0² + 1² + 1² + 1²) = √3
similarity = 2 / (√3 × √3) = 2/3 ≈ 0.67
```

#### 3.3.3 推荐算法实现（伪代码）

```python
# 基于用户的协同过滤推荐算法

def recommend_for_user(user_id, limit=10):
    """
    为用户推荐房源
    """
    # 1. 获取用户的浏览历史
    user_browsed = get_user_browse_history(user_id)
    
    # 2. 找到相似用户
    similar_users = find_similar_users(user_id, top_k=20)
    
    # 3. 收集相似用户浏览过的房源
    candidate_houses = {}
    for similar_user, similarity in similar_users:
        browsed = get_user_browse_history(similar_user.id)
        for house in browsed:
            if house not in user_browsed:  # 排除已浏览
                if house not in candidate_houses:
                    candidate_houses[house] = 0
                # 累加相似度作为推荐分数
                candidate_houses[house] += similarity
    
    # 4. 按分数排序，返回Top N
    recommendations = sorted(
        candidate_houses.items(), 
        key=lambda x: x[1], 
        reverse=True
    )[:limit]
    
    return recommendations


def find_similar_users(user_id, top_k=20):
    """
    找到与目标用户最相似的K个用户
    """
    target_user_vector = build_user_vector(user_id)
    similarities = []
    
    # 遍历所有其他用户
    for other_user in get_all_users():
        if other_user.id == user_id:
            continue
        
        other_user_vector = build_user_vector(other_user.id)
        
        # 计算余弦相似度
        similarity = cosine_similarity(target_user_vector, other_user_vector)
        
        if similarity > 0:  # 只保留有相似性的用户
            similarities.append((other_user, similarity))
    
    # 按相似度排序，返回Top K
    similarities.sort(key=lambda x: x[1], reverse=True)
    return similarities[:top_k]


def build_user_vector(user_id):
    """
    构建用户向量（基于浏览历史）
    """
    all_houses = get_all_houses()
    browsed_houses = get_user_browse_history(user_id)
    
    # 构建0-1向量：浏览过为1，未浏览为0
    vector = []
    for house in all_houses:
        vector.append(1 if house in browsed_houses else 0)
    
    return vector


def cosine_similarity(vec_a, vec_b):
    """
    计算余弦相似度
    """
    dot_product = sum(a * b for a, b in zip(vec_a, vec_b))
    norm_a = math.sqrt(sum(a * a for a in vec_a))
    norm_b = math.sqrt(sum(b * b for b in vec_b))
    
    if norm_a == 0 or norm_b == 0:
        return 0
    
    return dot_product / (norm_a * norm_b)
```

#### 3.3.4 混合推荐策略

```python
def hybrid_recommend(user_id, limit=10):
    """
    混合推荐策略：结合协同过滤、内容推荐和热门推荐
    """
    recommendations = []
    
    # 1. 协同过滤推荐（权重：0.5）
    cf_recs = collaborative_filtering_recommend(user_id, limit * 2)
    for house, score in cf_recs:
        recommendations.append({
            'house_id': house.id,
            'score': score * 0.5,
            'method': 'collaborative_filtering'
        })
    
    # 2. 内容推荐（权重：0.3）
    content_recs = content_based_recommend(user_id, limit * 2)
    for house, score in content_recs:
        # 如果已存在，累加分数；否则新增
        existing = next((r for r in recommendations if r['house_id'] == house.id), None)
        if existing:
            existing['score'] += score * 0.3
        else:
            recommendations.append({
                'house_id': house.id,
                'score': score * 0.3,
                'method': 'content_based'
            })
    
    # 3. 热门推荐（权重：0.2）
    popular_recs = popularity_based_recommend(limit * 2)
    for house, score in popular_recs:
        existing = next((r for r in recommendations if r['house_id'] == house.id), None)
        if existing:
            existing['score'] += score * 0.2
        else:
            recommendations.append({
                'house_id': house.id,
                'score': score * 0.2,
                'method': 'popularity_based'
            })
    
    # 4. 按综合分数排序
    recommendations.sort(key=lambda x: x['score'], reverse=True)
    
    return recommendations[:limit]


def content_based_recommend(user_id, limit=10):
    """
    基于内容的推荐：分析用户偏好
    """
    # 1. 分析用户浏览历史，提取偏好特征
    browse_history = get_user_browse_history(user_id)
    
    # 统计城市偏好
    city_preference = {}
    for house in browse_history:
        city = house.city
        city_preference[city] = city_preference.get(city, 0) + 1
    
    # 统计价格偏好
    prices = [house.price for house in browse_history]
    avg_price = sum(prices) / len(prices) if prices else 0
    price_range = (avg_price * 0.7, avg_price * 1.3)  # ±30%
    
    # 2. 查询符合偏好的房源
    candidate_houses = []
    for house in get_all_active_houses():
        if house in browse_history:
            continue  # 排除已浏览
        
        score = 0
        
        # 城市匹配加分
        if house.city in city_preference:
            score += city_preference[house.city] * 10
        
        # 价格匹配加分
        if price_range[0] <= house.price <= price_range[1]:
            score += 20
        
        # 评分加分
        score += house.avg_rating * 5
        
        candidate_houses.append((house, score))
    
    # 3. 排序返回
    candidate_houses.sort(key=lambda x: x[1], reverse=True)
    return candidate_houses[:limit]


def popularity_based_recommend(limit=10):
    """
    基于热度的推荐：评分高、订单多的房源
    """
    houses = get_all_active_houses()
    
    # 计算热度分数
    scored_houses = []
    for house in houses:
        # 热度 = 平均评分 × 10 + 订单数量
        popularity_score = house.avg_rating * 10 + house.order_count
        scored_houses.append((house, popularity_score))
    
    # 排序返回
    scored_houses.sort(key=lambda x: x[1], reverse=True)
    return scored_houses[:limit]
```

### 3.4 推荐效果评估

**评估指标**：
1. **准确率（Precision）**：推荐的房源中，用户实际感兴趣的比例
2. **召回率（Recall）**：用户感兴趣的房源中，被推荐的比例
3. **覆盖率（Coverage）**：推荐系统能够推荐的房源占总房源的比例
4. **多样性（Diversity）**：推荐结果的多样化程度

```java
@Service
public class BrowseHistoryService {

    @Autowired
    private BrowseHistoryRepository browseHistoryRepository;

    /**
     * 记录浏览行为
     * @param userId 用户ID
     * @param houseId 房源ID
     * @param duration 浏览时长（秒）
     */
    @Transactional
    public void recordBrowse(Long userId, Long houseId, Integer duration) {
        BrowseHistory history = new BrowseHistory();
        history.setUserId(userId);
        history.setHouseId(houseId);
        history.setBrowseTime(LocalDateTime.now());
        history.setBrowseDuration(duration);
        
        browseHistoryRepository.save(history);
    }

    /**
     * 获取用户最近的浏览记录
     * @param userId 用户ID
     * @param limit 数量限制
     */
    public List<BrowseHistory> getRecentBrowseHistory(Long userId, int limit) {
        List<BrowseHistory> all = browseHistoryRepository.findRecentByUserId(userId);
        return all.size() > limit ? all.subList(0, limit) : all;
    }

    /**
     * 获取用户最近N天的浏览记录
     */
    public List<BrowseHistory> getRecentDaysBrowseHistory(Long userId, int days) {
        LocalDateTime afterTime = LocalDateTime.now().minusDays(days);
        return browseHistoryRepository.findByUserIdAndBrowseTimeAfter(userId, afterTime);
    }

    /**
     * 获取最近浏览的房源详情（用于展示）
     */
    public List<Map<String, Object>> getRecentBrowsedHouses(Long userId, int limit) {
        List<BrowseHistory> history = getRecentBrowseHistory(userId, limit * 2);
        
        List<Map<String, Object>> result = new ArrayList<>();
        Set<Long> addedHouseIds = new HashSet<>();
        
        for (BrowseHistory browse : history) {
            if (result.size() >= limit) {
                break;
            }
            
            Long houseId = browse.getHouseId();
            
            // 去重：避免重复浏览的房源多次出现
            if (addedHouseIds.contains(houseId)) {
                continue;
            }
            
            House house = houseRepository.findById(houseId).orElse(null);
            if (house != null && "ACTIVE".equals(house.getStatus())) {
                Map<String, Object> houseData = new HashMap<>();
                houseData.put("houseId", house.getId());
                houseData.put("title", house.getTitle());
                houseData.put("price", house.getPrice());
                houseData.put("city", house.getCity());
                
                // 获取封面图片
                List<HouseImage> images = 
                    houseImageRepository.findByHouseIdOrderBySortOrderAsc(houseId);
                if (!images.isEmpty()) {
                    houseData.put("coverImage", images.get(0).getImageUrl());
                }
                
                // 获取平均评分
                Double avgRating = reviewRepository.getAverageRatingByHouseId(houseId);
                houseData.put("rating", avgRating != null ? avgRating : 0.0);
                
                result.add(houseData);
                addedHouseIds.add(houseId);
            }
        }
        
        return result;
    }

    /**
     * 清空用户的浏览记录
     */
    @Transactional
    public void clearUserHistory(Long userId) {
        browseHistoryRepository.deleteByUserId(userId);
    }
}
```

### 5.2 Cron表达式详解

**格式**：`秒 分 时 日 月 星期 [年]`

**常用示例**：
```
0 0 2 * * ?        每天凌晨2点执行
0 */30 * * * ?     每30分钟执行一次
0 0 12 * * ?       每天中午12点执行
0 0 0 1 * ?        每月1号凌晨执行
```

- `-`：范围（如：10-12）
- `,`：列举（如：MON,WED,FRI）
- `/`：增量（如：0/15表示每15分钟）

```java
@PostMapping("/image")
public String uploadHouseImages(
        @RequestParam Long houseId,
        @RequestParam("files") MultipartFile[] files
) throws Exception {

    // 1. 验证文件是否为空
    if (files == null || files.length == 0) {
        throw new RuntimeException("请上传图片");
    }

    // 2. 设置上传目录（按房源ID分类）
    String uploadDir = "D:/wlh/homestay_upload/house/" + houseId + "/";

    // 3. 创建目录（如果不存在）
    File dir = new File(uploadDir);
    if (!dir.exists()) {
        dir.mkdirs();
    }

    List<HouseImage> images = new ArrayList<>();

    // 4. 遍历上传的文件
    for (int i = 0; i < files.length; i++) {
        MultipartFile file = files[i];
        if (file.isEmpty()) continue;

        // 5. 生成唯一文件名（UUID + 扩展名）
        String originalFilename = file.getOriginalFilename();
        String extension = originalFilename.substring(originalFilename.lastIndexOf("."));
        String fileName = UUID.randomUUID() + extension;

        // 6. 保存文件到磁盘
        file.transferTo(new File(uploadDir + fileName));

        // 7. 保存图片记录到数据库
        HouseImage img = new HouseImage();
        img.setHouseId(houseId);
        img.setImageUrl("/house/" + houseId + "/" + fileName);  // 相对路径
        img.setSortOrder(i + 1);  // 排序顺序

        images.add(img);
    }

    // 8. 批量保存到数据库
    houseImageRepository.saveAll(images);

    return "ok";
}
```

## 1. 项目概述

### 1.1 项目背景
随着共享经济的发展和旅游业的繁荣，民宿预订平台成为连接房东与租客的重要桥梁。本系统旨在打造一个功能完善、用户体验良好的民宿预订管理平台。

### 1.2 项目目标
- 为普通用户提供便捷的民宿搜索、预订、评价功能
- 为房东提供房源管理、订单处理、收益统计等功能
- 为管理员提供平台监控、用户管理、房源审核等功能
- 实现智能推荐、AI客服等创新功能

### 1.3 系统角色
- **普通用户（USER）**：浏览房源、预订、评价、管理个人信息
- **房东（HOST）**：发布房源、管理订单、查看收益、回复评价
- **管理员（ADMIN）**：审核房源、管理用户、监控订单、数据统计

// JWT Token 生成
public String generateToken(Long userId, String username, String role) {
    return Jwts.builder()
        .setSubject(username)
        .claim("userId", userId)
        .claim("role", role)
        .setIssuedAt(new Date())
        .setExpiration(new Date(System.currentTimeMillis() + 86400000))
        .signWith(SignatureAlgorithm.HS512, SECRET_KEY)
        .compact();
}

public class OrderScheduler {
    
    @Scheduled(cron = "0 0 2 * * ?") // 每天凌晨2点执行
    public void autoCompleteOrders() {
        LocalDate yesterday = LocalDate.now().minusDays(1);
        List<Order> orders = orderRepository
            .findByStatusAndCheckOutDateBefore("CONFIRMED", yesterday);
        
        orders.forEach(order -> {
            order.setStatus("COMPLETED");
            orderRepository.save(order);
        });
    }
}

@Bean
public CorsFilter corsFilter() {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowCredentials(true);
    config.addAllowedOriginPattern("*");
    config.addAllowedHeader("*");
    config.addAllowedMethod("*");
    
    UrlBasedCorsConfigurationSource source = 
        new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", config);
    
    return new CorsFilter(source);
}

public boolean isDateAvailable(Long houseId,

    List<Order> conflicts = orderRepository
        .findConflictingOrders(houseId, checkIn, checkOut);
    return conflicts.isEmpty();
}

// SQL查询
@Query("SELECT o FROM Order o WHERE o.houseId = :houseId " +

### 6.6 前端路由守卫
**问题描述**：需要保护需要登录的页面，验证用户权限

**解决方案**：
```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  const role = localStorage.getItem('role')
  
  // 检查是否需要登录
  if (to.matched.some(record => record.meta.requiresAuth)) {
    if (!token) {
      alert('请先登录！')
      next('/user')
      return
    }
    
    // 检查角色权限
    if (to.path.startsWith('/host') && role !== 'ROLE_HOST') {
      alert('您没有房东权限！')
      next('/user')
      return
    }
    
    if (to.path.startsWith('/admin') && role !== 'ROLE_ADMIN') {
      alert('您没有管理员权限！')
      next('/user')
      return
    }
  }
  
  next()
})
```

### 6.7 数据统计性能优化
**问题描述**：管理员仪表板需要统计大量数据，查询效率低

**解决方案**：
1. **使用索引**：在常用查询字段上建立索引
2. **SQL优化**：使用聚合函数，减少数据传输
3. **缓存策略**：对统计数据进行缓存（可选）

```sql
-- 优化前：查询所有订单再统计
SELECT * FROM `order` WHERE status = 'COMPLETED'

-- 优化后：直接统计
SELECT 
  COUNT(*) as total_orders,
  SUM(total_price) as total_revenue,
  AVG(total_price) as avg_price
FROM `order`
WHERE status = 'COMPLETED'
```

### 8.2 可能的提问及回答

**Q1：为什么选择前后端分离架构？**
A：前后端分离有以下优势：
- 开发效率高，前后端可并行开发
- 职责清晰，便于维护和扩展
- 前端可以灵活选择技术栈
- 便于实现多端适配（Web、移动端）

**Q2：JWT和Session的区别？为什么选择JWT？**
A：
- Session：服务器端存储，占用服务器内存，不利于分布式部署
- JWT：无状态，服务器不存储，适合分布式系统
- 选择JWT是因为：无状态、可扩展、跨域友好

**Q3：如何保证订单不会重复预订？**
A：通过数据库查询检测日期冲突：
```sql
查询条件：同一房源 AND 状态为待确认/已确认 
AND (新订单入住日期 <= 已有订单退房日期 
AND 新订单退房日期 >= 已有订单入住日期)
如果查询结果为空，则日期可用
```

**Q4：如何实现智能推荐？**
A：基于用户行为的协同过滤：
1. 分析用户浏览历史
2. 统计偏好城市和价格区间
3. 查询符合条件的房源
4. 按评分和热度排序
5. 返回Top N推荐

**Q5：系统的安全性如何保证？**
A：
- 密码使用BCrypt加密存储
- 使用JWT Token认证
- Spring Security权限控制
- SQL参数化查询防止注入
- 前端路由守卫保护页面

**Q6：如果用户量增大，如何优化性能？**
A：
- 引入Redis缓存热点数据
- 数据库读写分离
- 使用CDN加速静态资源
- 图片压缩和懒加载
- 数据库索引优化
- 引入消息队列处理异步任务

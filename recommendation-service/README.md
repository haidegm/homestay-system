# 民宿推荐系统

## 概述

本推荐系统采用混合推荐算法，结合协同过滤和基于内容的推荐方法，为用户提供个性化的民宿推荐。

## 推荐算法原理

### 1. 混合推荐策略

系统采用两种推荐方法的加权组合：

- **协同过滤（60%权重）**：基于"订了A的人也订了B"的逻辑
- **基于内容推荐（40%权重）**：基于房源属性相似度

最终推荐分数 = 协同过滤分数 × 0.6 + 基于内容分数 × 0.4

### 2. 协同过滤推荐

#### 核心思想
找出与当前用户有相似行为的其他用户，推荐这些相似用户喜欢但当前用户没有交互过的房源。

#### 实现步骤

1. **获取用户交互记录**
   - 订单记录（已确认、已完成）
   - 收藏记录
   - 浏览记录
   - 评价记录

2. **寻找相似用户**
   - 找出也交互过相同房源的其他用户
   - 计算相似度：相似度 = 共同交互房源数 / 当前用户交互房源数
   - 选取相似度最高的前20个用户

3. **统计相似用户喜欢的房源**
   - 获取相似用户的所有交互记录
   - 根据交互类型赋予不同权重：
     - 已完成订单：5.0分
     - 收藏：3.0分
     - 浏览：2.0分
   - 推荐分数 = 用户相似度 × 行为权重

4. **排除已交互房源**
   - 不推荐用户已经订购、收藏、浏览或评价过的房源

#### 示例场景
```
用户A订购了房源1、2、3
用户B订购了房源1、2、4
用户C订购了房源1、5

相似度计算：
- 用户B与用户A的相似度 = 2/3 = 0.67（共同订购了房源1、2）
- 用户C与用户A的相似度 = 1/3 = 0.33（共同订购了房源1）

推荐给用户A：
- 房源4（来自用户B，分数 = 0.67 × 5.0 = 3.35）
- 房源5（来自用户C，分数 = 0.33 × 5.0 = 1.65）
```

### 3. 基于内容推荐

#### 核心思想
找出与用户浏览/订购/收藏过的房源相似的其他房源。

#### 相似度计算因素

1. **城市相同**：+1.0分（最重要）
2. **价格相近**（±30%）：+0.5分（根据价格差异递减）
3. **容纳人数相同**：+0.3分
4. **床位数相同**：+0.2分

#### 用户行为权重

根据用户行为类型和时间赋予不同权重：

| 行为类型 | 权重 | 说明 |
|---------|------|------|
| 评价 | 5.0 × 评分 | 最高权重，反映真实体验 |
| 已完成订单 | 4.0 | 已入住，满意度高 |
| 已确认订单 | 3.5 | 即将入住 |
| 收藏 | 3.2 | 明确的兴趣表达 |
| 1天内浏览 | 4.5 | 最近浏览，权重很高 |
| 3天内浏览 | 4.0 | 近期浏览 |
| 7天内浏览 | 3.5 | 一周内浏览 |
| 30天内浏览 | 3.0 | 一个月内浏览 |
| 更早浏览 | 2.5 | 较早浏览 |

#### 时间衰减机制

浏览记录采用时间衰减机制，最近浏览的房源权重更高，确保推荐的实时性。

#### 推荐计算

对于每个候选房源：
1. 计算与用户所有交互房源的相似度
2. 推荐分数 = Σ(相似度 × 用户行为权重)
3. 排除用户已交互的房源

#### 示例场景
```
用户浏览了杭州的房源A（价格300元，1天前）
系统推荐：
- 杭州的房源B（价格280元）
  相似度 = 1.0（同城市）+ 0.47（价格相近）= 1.47
  推荐分数 = 1.47 × 4.5（1天内浏览）= 6.62
```

### 4. 热门房源补充

当推荐结果不足时，系统会补充热门房源：

热门度分数 = 订单数 × 0.7 + 平均评分 × 0.3

这确保新用户或交互较少的用户也能获得推荐。

## 推荐流程

```
用户请求推荐
    ↓
协同过滤推荐（找相似用户喜欢的房源）
    ↓
基于内容推荐（找相似房源）
    ↓
混合计算（60% + 40%）
    ↓
结果不足？ → 补充热门房源
    ↓
排序并返回Top N
```

## API接口

### 1. 用户个性化推荐
```
GET /api/recommend/user/{user_id}?limit=10
```

返回为指定用户推荐的房源列表。

### 2. 相似房源推荐
```
GET /api/recommend/similar/{house_id}?limit=10
```

返回与指定房源相似的其他房源。

### 3. 热门房源
```
GET /api/recommend/popular?limit=10
```

返回平台热门房源。

### 4. 健康检查
```
GET /health
```

检查服务是否正常运行。

## 技术特点

1. **实时计算**：不需要离线训练，每次请求实时计算推荐结果
2. **冷启动处理**：新用户或无交互记录时，推荐热门房源
3. **时间衰减**：浏览记录考虑时间因素，最近浏览权重更高
4. **多维度相似度**：综合考虑城市、价格、容纳人数等多个维度
5. **混合策略**：结合协同过滤和基于内容的优势

## 数据来源

推荐系统使用以下数据表：

- `order`：订单记录
- `wishlist`：收藏记录
- `browse_history`：浏览记录
- `review`：评价记录
- `house`：房源信息

创建 `.env` 文件：

```env

DB_USER=root

```

## 运行服务

```bash
# 安装依赖
pip install flask flask-cors pymysql python-dotenv

# 启动服务
python app.py
```

服务将在 `http://localhost:5000` 启动。

## 优化方向

1. **缓存机制**：对热门推荐结果进行缓存，减少数据库查询
2. **A/B测试**：测试不同权重配置的效果
3. **深度学习**：引入神经网络模型提升推荐准确度
4. **实时反馈**：根据用户点击、停留时间等实时调整推荐
5. **多样性优化**：避免推荐结果过于集中在某个城市或价格区间
